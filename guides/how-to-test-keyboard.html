How to Test a Keyboard Как проверить клавиатуру

Fast checks for every key and common problems. Use the online tester to see key highlights and codes. Быстрая проверка всех клавиш и частых проблем. В онлайн‑тестере вы увидите подсветку нажатых клавиш и коды.

Start keyboard test Начать тест If a key fails Если клавиша не работает

1) Test letters and numbers 1) Проверьте буквы и цифры

  1. Open the tester. Откройте тестер.
  2. Press A–Z slowly, then 0–9. Нажмите A–Z по очереди, затем 0–9.
  3. Look for keys that don’t light up or light up late. Ищите клавиши, которые не загораются или срабатывают с задержкой.

Tip: A key that triggers twice can be a switch issue or debris under the keycap. Клавиша, которая «двойнит», может быть грязной или с проблемой переключателя.

2) Test modifiers (Shift, Ctrl, Alt, Win/Command) 2) Проверьте модификаторы (Shift, Ctrl, Alt, Win/Command)

Hold each modifier for a second to confirm it registers reliably. Then test common combos like Ctrl+C, Ctrl+V, and Alt+Tab. Зажмите каждый модификатор на 1 секунду, чтобы убедиться, что он стабильно регистрируется. Затем проверьте комбинации Ctrl+C, Ctrl+V, Alt+Tab.

  • Windows: Ctrl, Alt, Win Windows: Ctrl, Alt, Win
  • Mac: Control, Option, Command Mac: Control, Option, Command

3) Test special keys 3) Проверьте специальные клавиши

  • Backspace, Enter, SpaceBackspace, Enter, Space
  • Arrow keys + Home/End/PageUp/PageDownСтрелки + Home/End/PageUp/PageDown
  • F-keys (F1–F12) and Fn combinations (if your keyboard has Fn)F‑клавиши (F1–F12) и сочетания с Fn (если есть Fn)
  • Media keys (volume, play/pause) if availableМедиа‑клавиши (громкость, play/pause), если есть

4) Check for ghosting / rollover (optional) 4) Проверьте гостинг / rollover (опционально)

Ghosting happens when certain key combinations cannot be detected at the same time. If you play games or type fast, test common combos you use. Гостинг — когда некоторые сочетания не регистрируются одновременно. Если вы играете или быстро печатаете, проверьте ваши привычные комбинации.

FAQ

Does this tester record my typing? Тестер записывает мои нажатия?

No. The tool reads key events in your browser to show which keys are pressed. See the Privacy page for details. Нет. Инструмент читает события клавиатуры в браузере только чтобы подсветить нажатые клавиши. Подробности — на странице конфиденциальности.

Why do I see “event.code” and “keyCode”? Почему показываются “event.code” и “keyCode”?

They help diagnose layout issues, remaps, and app-specific shortcuts. “event.code” describes the physical key; “keyCode” is legacy but still used in some apps. Это помогает диагностировать раскладку, переназначения и сочетания. “event.code” — физическая клавиша, “keyCode” — старый формат, но иногда ещё используется.

One key works in the tester but not in my app — why? Клавиша работает в тестере, но не работает в приложении — почему?

The app might capture the key as a shortcut, or a remapping tool/IME is changing input. Try another app and check keyboard settings. Приложение может перехватывать клавишу как хоткей, или ввод меняют ремап‑утилиты/IME. Проверьте в другом приложении и настройки клавиатуры.
